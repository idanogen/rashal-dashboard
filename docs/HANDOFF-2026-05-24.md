## ✅ מה הסתיים היום (הכל live בייצור)

### 1. WhatsApp via heyy.io — תשתית מלאה (במצב דמו)
- דף `/whatsapp` עם 4 טאבים: נכנסות, יוצאות, Templates, תזכורת יומית
- **דמו מלא מקצה-לקצה** — אפשר לסמלץ שליחה ותשובת לקוח בלי heyy אמיתי
- 4 templates רשומים בקוד: `delivery_reminder`, `schedule_request`, `schedule_coordination`, `team_notification`
- 3 API endpoints: `/api/heyy-send`, `/api/heyy-webhook`, `/api/cron-daily-reminders`
- Vercel cron מתוזמן ל-06:00 UTC כל יום (08-09 שעון ישראל)

### 2. תיאום משלוחים על היומן
- כפתור 📱 **תיאום** על כל stop בדף `/routes`
- Dialog עם שעות (start/end), הערה חופשית, ו-2 כפתורי פעולה: WhatsApp / טלפוני
- Badge חזותי על הכרטיס + 🎉 confetti כשהלקוח אישר

### 3. ניהול משתמשים
- דף חדש `/admin/users` (קישור 👥 בנאב — רק לאדמין)
- יצירת משתמשים עם סיסמה זמנית או הזמנה ב-email
- 4 תפקידים: admin / dispatcher / driver / viewer
- קישור user→driver_name (linked_driver) למשתמשי נהג

### 4. אבטחה (RLS לפי תפקיד)
- כל הטבלאות הוגנו ב-RLS מבוסס תפקיד
- driver רואה רק את ה-stops שלו, admin/dispatcher רואים הכל, viewer קריאה בלבד

### 5. מסך נהג ייעודי
- `/driver` — UI mobile-first בלי nav אדמין
- 3 טאבים (היום/מחר/השבוע), כרטיסי stop גדולים, ניווט Waze, tap-to-call, סימון בוצע/לא-בוצע
- נהג מועבר אוטומטית מ-`/` ל-`/driver`

1. היכנס ל-`/admin/users`
2. **"הוסף משתמש"** → צור עם סיסמה זמנית:
   - אימייל: `test-driver@rashal.co.il` (או מה שמתאים)
   - שם מלא: `נהג טסט`
   - תפקיד: **נהג**
   - בחר נהג מקושר: `רודי דויד` (או הנהג החיצוני)
3. העתק את הסיסמה הזמנית
4. פתח דפדפן incognito → התחבר עם הפרטים
5. ודא שאתה נוחת ב-`/driver` (לא ב-`/`)
6. ודא שאתה רואה רק את ה-stops של הנהג המקושר
7. נסה ללחוץ כל כפתור: ניווט Waze, התקשרות, תיאום WhatsApp, בוצע/לא בוצע
8. נסה לגשת ל-`/admin/users` → אמור להחזיר אותך ל-`/driver`

### ב. (אופציונלי) להתחיל את התהליך מול heyy.io
- אם תרצה להפעיל וואטסאפ אמיתי: **פתח חשבון ב-https://heyy.io**
- חבר ערוץ WhatsApp Business (1-3 ימים אם נדרש אימות עסק חדש מול Meta)
- **תוך כדי** — אפשר להגיש את ה-templates לאישור Meta (24-48h אישור)
  - פירוט מלא ב-`docs/heyy-setup.md` — שלב 3
- אחרי שיש חשבון:
  1. הוסף ENV vars ב-Vercel: `HEYY_API_KEY`, `HEYY_CHANNEL_ID`, `HEYY_WEBHOOK_SECRET`
  2. החלף `DEMO-*` ב-UUIDs האמיתיים ב-`src/lib/heyy/templates.ts`
  3. שנה `HEYY_MODE=real` + `VITE_HEYY_MODE=real` ב-Vercel
  4. בdeash UI: הגדר webhook → `https://rashal-dashboard.vercel.app/api/heyy-webhook`
  5. Deploy

### ג. (אופציונלי, ביטחוני) רוטציה של Service Role Key
- ה-key הנוכחי הוא Legacy JWT שדלף בהיסטוריית הצ'אט
- ב-Supabase → Settings → API Keys → טאב "Publishable and secret API keys" → Create new secret key
- שם את החדש ב-Vercel (`vercel env rm SUPABASE_SERVICE_ROLE_KEY production` ואז `vercel env add`)
- Disable את הישן בטאב Legacy

### ד. תיקון 4 שגיאות TS קיימות (לא דחוף)
`src/pages/ServiceCallsPage.tsx` שורות 291, 329, 538 — `address` לא קיים ב-`ServiceCall` type. לא קשור לעבודה שלי — תיקון נפרד צריך.
